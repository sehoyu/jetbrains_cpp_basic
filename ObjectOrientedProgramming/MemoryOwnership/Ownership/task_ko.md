C++에서, 소유권(ownership)은 메모리, 파일, 네트워크 연결, 객체 등 특정 자원의 생명 주기를 관리하는 책임을 의미합니다.  
C++의 소유권 모델은 크게 두 가지로 분류될 수 있습니다:  
객체 필드에 저장된 하위 객체의 소유권과  
포인터를 사용하여 동적으로 생성된 객체의 소유권입니다.

C++에서는 클래스가 하위 객체를 나타내는 멤버 필드를 포함하는 경우가 많습니다.  
이러한 하위 객체는 일반적으로 부모 객체에 의해 소유됩니다.  
부모 객체는 이들의 생성자와 소멸자를 호출할 책임이 있으며,  
이들이 점유하는 메모리는 부모 객체의 생명 주기에 의해 자동으로 관리됩니다.

<div class="hint">

생성자와 소멸자는 특정한 순서로 호출되며,  
특히 소멸 순서는 초기화 순서의 반대라는 점에 유의하세요.  
예를 들어, 다음과 같은 클래스 계층 구조를 고려해 봅시다:

```c++
class Motherboard { /* ... */ };
class Processor { /* ... */ };
class RandomAccessMemory { /* ... */ };

class Device { /* ... */ };

class Laptop : public Device {
private:
    // 필드에 저장된 객체의 소유권은 
    // 자동으로 관리됩니다
    Motherboard motherboard;
    Processor processor;  
    RandomAccessMemory ram;
public:
    /* ... */
};
```

`Laptop` 객체의 생성 순서는 다음과 같습니다:  
1. 기본 클래스 `Device` 생성자가 가장 먼저 호출됩니다.  
2. 그 다음으로, 하위 객체의 생성자가 호출됩니다:  
   * 필드 `motherboard`의 `Motherboard` 생성자,  
   * 필드 `processor`의 `Processor` 생성자,  
   * 필드 `ram`의 `RandomAccessMemory` 생성자.  
3. 마지막으로 `Laptop` 생성자가 호출됩니다.

소멸 순서는 그 반대입니다:  
1. 먼저 파생 클래스 소멸자 `Laptop`이 호출됩니다.  
2. 그 후 하위 객체의 소멸자가 호출됩니다:  
   * `RandomAccessMemory` 소멸자,  
   * `Processor` 소멸자,  
   * `Motherboard` 소멸자.  
3. 마지막으로 기본 클래스 `Device` 소멸자가 호출됩니다.

</div>

기본적으로 포인터로 선언된 필드에 저장된 객체는 비소유(non-owned)로 간주됩니다.  
클래스가 객체에 대한 포인터를 포함할 경우,  
해당 객체의 생명 주기와 저장 기간 관리는 클래스 외부에서 이루어져야 합니다.  
이 사실을 제대로 이해하는 것은 메모리 누수와 정의되지 않은 동작을 방지하기 위해 매우 중요합니다.  
포인터가 할당될 때 소유권은 자동으로 전달되지 않기 때문입니다.

```c++
class Student {
private:
    // 기본적으로 비소유 상태
    Laptop* laptop;
public:
    explicit Student(Laptop* laptop) 
        : laptop(laptop) {}
        
    // 소멸자는 기본적으로 
    // laptop 필드가 가리키는 객체를 
    // 소멸시키지 않습니다.    
    ~Student() = default;
};
```

위 예제에서 `Student` 클래스는  
동적으로 할당된 `Laptop` 객체에 대한 포인터를 포함하고 있습니다.  
`Laptop` 객체에 대한 소유권은  
생성자에서 `Student` 클래스에 할당될 수도 있고, 그렇지 않을 수도 있습니다.  
만약 `laptop` 객체의 소유권을 생성자에서  
`Student`가 가져가야 한다는 설계 의도를 가진다면,  
해당 객체를 소멸자에서 수동으로 삭제해야 한다는 점을 개발자가 보장해야 합니다  
(예: `delete` 연산자를 사용하는 방식).

포인터 필드에 저장된 객체가 기본적으로 비소유로 간주되는 이유는  
메모리 관리를 더욱 명시적으로 제어할 수 있고,  
유연성과 효율성을 제공하기 위함입니다.

소유권을 이전해야 하는 경우에는,  
[`스마트 포인터`](https://en.wikipedia.org/wiki/Smart_pointer)  
예를 들면, `std::unique_ptr`와 `std::shared_ptr` 같은  
스마트 포인터를 사용할 수 있습니다.  
스마트 포인터는 메모리를 자동으로 관리하며,  
메모리와 관련된 문제의 위험을 줄여줍니다.  
이 스마트 포인터들은 C++ 표준 라이브러리에서 정의된 특수 클래스입니다.  
이들은 기본 포인터처럼 행동하며,  
역참조와 같은 동일한 연산도 지원하지만,  
추가적으로 특정한 소유권 개념을 제공합니다.  
스마트 포인터에 대해서는 이후 몇 강의에서 자세히 다룰 것입니다.
