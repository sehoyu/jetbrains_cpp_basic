C++ 언어의 소유권 모델에 대한 설명을 마무리하면서, 우리는 몇 가지 필수적인 개념과 관용구에 대해 논의할 필요가 있습니다.

[_자원 획득은 초기화_](https://en.cppreference.com/w/cpp/language/raii) 또는 줄여서 __RAII__는 객체의 수명과 자원(예: 동적으로 할당된 메모리 블록, 열린 파일, 네트워크 연결 등)을 연결하는 프로그래밍 기법입니다. 이 관용구에 따르면: 

1. 클래스의 생성자는 객체의 수명 동안 소유할 모든 자원을 할당하고 초기화해야 합니다.
2. 클래스의 복사 생성자와 복사 대입 연산자는 자원을 복사해야 합니다. 
   자원이 복사 불가능한 경우, 이 생성자는 삭제되어야 합니다.
3. 클래스의 이동 생성자와 이동 대입 연산자는 자원의 소유권을 이전해야 합니다.
4. 클래스의 소멸자는 객체가 소유한 모든 자원을 해제해야 합니다.

[**5의 법칙**](https://en.cppreference.com/w/cpp/language/rule_of_three)은 관련된 개념입니다. 이는 클래스가 맞춤 정의가 필요한 다음 메서드:

1. 소멸자
2. 복사 생성자
3. 복사 대입 연산자
4. 이동 생성자
5. 이동 대입 연산자

중 하나라도 포함하는 경우, 다섯 개 모두 정의해야 한다고 규정합니다.

클래스가 이러한 메서드 중 아무것도 정의하지 않으면 컴파일러가 기본 구현을 생성한다는 점에 유의하세요:

* 기본 복사 생성자/대입 연산자는 얕은 복사를 수행하며, 멤버 필드의 복사 생성자/대입 연산자를 호출합니다.
* 기본 이동 생성자/대입 연산자는 클래스 멤버를 요소별로 이동하며, 멤버 필드의 이동 생성자/대입 연산자를 호출합니다.
* 추가적으로, 맞춤 생성자가 없는 경우 기본 생성자가 자동으로 생성되며, 이는 모든 멤버 필드를 기본값으로 초기화합니다.

`= default` 문법을 사용하여 기본 구현 생성 강제를 할 수 있습니다:

```c++
class X {
public:
    X() = default;
    X(const X&) = default;
    X(X&&) = default;
    
    X& operator=(const X&) = default;
    X& operator=(X&&) = default;
    /* ... */
};
```

이는 "다섯 가지" 메서드 중 하나를 사용자 정의로 구현했지만 나머지는 기본 구현을 사용하고 싶을 때 유용합니다.

반대로, `= delete` 문법을 사용하여 이러한 메서드의 자동 생성을 명시적으로 금지할 수 있습니다:

```c++
class X {
public:
    X() = delete;
    X(const X&) = delete;
    X(X&&) = delete;
    
    X& operator=(const X&) = delete;
    X& operator=(X&&) = delete;    
    /* ... */
};
```

예를 들어, 클래스가 복사 불가능해야 하는 경우 이는 유용할 수 있습니다. 이러한 시나리오에서는 이동 생성자와 대입 연산자를 정의하지만, 복사 생성자와 복사 대입 연산자는 삭제합니다.

<div class="hint">

[C++11 버전](https://en.cppreference.com/w/cpp/11) 이전에는 **5의 법칙**이 _3의 법칙_이라고 불렸습니다. 이는 C++11 이전에는 이동 의미론 기능이 없어 이동 생성자와 이동 대입 연산자를 정의할 수 없었기 때문입니다.

</div>

`int_array` 클래스를 다시 살펴보고, 이 클래스가 RAII와 5의 법칙에 어떻게 부합하는지 확인해 봅시다 (`include/int_array.hpp` 파일의 클래스 전체 정의를 참조하세요). 이 클래스는 동적으로 할당된 정수 배열이라는 자원을 관리합니다. 이 배열의 생명 주기는 `int_array` 객체의 수명과 연결되어 있습니다.

기본 생성자는 빈 배열을 생성합니다:

```c++
int_array()
    : data_(nullptr)
    , size_(0)
{};
```

기본 생성자를 다른 관점에서 보면, 이는 배열을 특별한 "빈" 상태로 생성합니다. 이는 나중에 유용하게 사용될 수 있습니다.

클래스의 또 다른 생성자는 주어진 크기의 배열을 실제로 메모리를 할당하여 생성합니다:

```c++
explicit int_array(std::size_t size)
    : data_(new int[size])
    , size_(size)
{};
```

따라서 소멸자는 이 메모리를 해제해야 합니다:

```c++
~int_array() {
    delete[] data_;
}
```

이제, 5의 법칙에 따라 이 클래스는 맞춤 복사 및 이동 생성자뿐 아니라 복사 및 이동 대입 연산자도 정의해야 합니다. 이전 강의에서 이미 이러한 생성자들을 본 바 있습니다:

```c++
int_array(const int_array& other)
        : data_(new int[other.size_])
        , size_(other.size_)
    {
        for (size_t i = 0; i < size_; ++i) {
            data_[i] = other.data_[i];
        }
    };

int_array(int_array&& other)
    : data_(other.data_)
    , size_(other.size_)
{
    other.data_ = nullptr;
    other.size_ = 0;
};
```

이동 생성자가 인수 객체를 "빈" 상태로 재설정하는 방식에 주목하세요. 나중에 이 "빈" 객체에 대해 소멸자를 호출해도 실제로 효과가 없습니다. 이는 자원의 소유권이 새 객체로 이전되었기 때문에 적절합니다.

다음으로, 대입 연산자를 정의해야 합니다. 복사 생성자와 복사 대입 연산자 간, 이동 생성자와 이동 대입 연산자 간의 코드 중복을 방지하기 위해 [__Copy-and-Swap 관용구__](https://en.wikibooks.org/wiki/More_C%2B%2B_Idioms/Copy-and-swap)를 사용할 수 있습니다.

이 관용구를 이용하면, 단일 버전의 대입 연산자만 정의하면 됩니다. 이 대입 연산자는 인수를 값으로 받습니다:

```c++
int_array& operator=(int_array other) {
    swap(other);
    return *this;
}
```

대입 연산자가 lvalue(`const int_array&`)와 함께 호출되는지 rvalue(`int_array&&`)와 함께 호출되는지에 따라 복사 생성자나 이동 생성자가 `other` 변수 내 데이터의 로컬 복사본(`int_array`)을 생성합니다. 그런 다음 `swap` 함수(아래 정의 참조)의 도움으로 대입 연산자는 현재 객체의 데이터를 로컬 복사본과 교환합니다. 임시 로컬 복사본은 파괴되며, 이 과정에서 이전 데이터를 해제합니다.

`swap` 함수는 두 객체 간 데이터를 교환하며, 기본 함수 `std::swap`을 사용하여 원시 타입 변수들을 교환합니다:

```c++
void swap(int_array& other) {
    std::swap(data_, other.data_);
    std::swap(size_, other.size_);
}
```

이렇게 해서 `int_array` 클래스는 _RAII_ 원칙을 구현하며, _5의 법칙_ 및 _Copy-and-Swap_ 관용구를 따릅니다.
