다양한 포인터 타입 간의 차이를 요약하기 위해, 아래 표를 살펴보겠습니다:

| 특징                                | `T*`                          | `std::unique_ptr<T>`                    | `std::shared_ptr<T>`                             | `std::weak_ptr<T>`                                         |
|-------------------------------------|-------------------------------|-----------------------------------------|--------------------------------------------------|------------------------------------------------------------|
| **소유권 의미**                     | 수동 소유권 관리              | 고유 소유권                             | 공유 소유권                                      | 비침습적 관찰자, 소유권 없음                                |
| **메모리 관리**                     | 수동 해제 필요                | 자동 해제                               | 자동 해제                                        | 메모리 해제 책임 없음                                       |
| **복사 가능 여부**                  | 복사 가능 (얕은 복사)         | 이동 가능 (소유권 전송)                  | 복사 가능 (공유 소유권)                           | 복사 가능 (공유 소유권)                                     |
| **참조 카운팅**                     | 없음                          | 없음                                    | 있음                                             | 있음                                                       |
| **순환 의존성 해결**                | 해당 없음                     | 해당 없음                               | `weak_ptr`를 사용해 해결                          | `weak_ptr`를 사용해 해결                                     |
| **사용 사례**                       | 저수준 메모리 조작            | 고유 소유권, 소유권 양도 불가           | 공유 소유권, 다중 참조                           | 공유 소유권을 관찰하되 라이프타임에 영향 없음               |
| **일반적인 사용 방식**              | 기본 포인터 사용              | 범위 기반 소유권, 메모리 누수 방지       | 공유 리소스 관리, 메모리 누수 방지               | 순환 의존성 해소, 공유 소유권 관찰                          |
| **스코프 종료 시 자동 정리**         | 없음                          | 있음                                    | 있음                                             | 해당 없음 (소유권 없음)                                      |
