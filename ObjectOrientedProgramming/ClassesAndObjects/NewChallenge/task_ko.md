이제 새로 습득한 지식을 활용하여 게임의 메커니즘을 확장하고, 새로운 종류의 게임 객체, 즉 블랙홀로 시각화된 적 객체를 추가해 보겠습니다.  
객체 지향 프로그래밍 기법을 사용하면 이 작업을 비교적 쉽게 수행할 수 있다는 것을 알게 될 것입니다.

적 객체는 다음과 같이 동작해야 합니다:
- 장면(scene)에서 무작위 방향으로 이동해야 하며, 주기적으로 그 방향을 변경해야 합니다.
- 충돌 시 별(star) 객체를 흡수해야 합니다.
- 적 객체가 접근할 때 별 객체의 상태는 `WARNED`로 변경되어야 합니다  
  (플레이어 객체가 접근할 때와 유사하게 동작합니다).
- 적 객체가 플레이어 객체와 충돌하면, 플레이어 객체의 상태는 `DESTROYED`로 변경되어  
  사실상 움직일 수 없게 되어야 합니다.

이 동작 구현을 진행해 보겠습니다.  
먼저, `EnemyObject` 클래스의 선언을 살펴보세요.  
이 클래스는 `CircleGameObject` 클래스를 상속받으므로, 해당하는 원형 형태와 행동이 이미 추가되어 있습니다.  
또한, `EnemyObject` 클래스의 `getKind()` 메서드는  
플레이어와 소비 가능한 객체(consumable)와는 구별되는 적 객체임을 나타내기 위해 `GameObjectKind::ENEMY` 값을 반환합니다.

`EnemyObject`의 이동 동작은 이미 일부 구현되어 있습니다.  
이를 위해 이 클래스 객체에 두 개의 새로운 필드를 추가했습니다:
- `velocity` 필드는 객체의 현재 속도의 방향과 속력을 저장하는 벡터입니다.  
- `updateTimer` 필드는 객체 속도가 마지막으로 업데이트된 이후 경과 시간을 저장합니다.

`getVelocity()` 메서드는 단순히 `velocity` 필드의 값을 반환합니다.  
`update(sf::Time delta)` 메서드는 객체 속도를 주기적으로 업데이트하는 역할을 합니다.  
이 메서드는 마지막 업데이트 이후 경과 시간을 인수로 받습니다.  
구현은 전체 경과 시간이 미리 정의된 시간 간격(1초)을 초과했는지 확인하고,  
초과했다면 `updateVelocity()` 메서드를 호출하여 새로운 무작위 속도를 생성하도록  
`velocity`를 업데이트하는 방식으로 이루어져 있습니다.  
이 메서드를 구현하는 것이 여러분의 과제입니다.

이 메서드는 무작위 방향 벡터를 생성하고, 속도 상수와 곱한 결과를 `velocity` 필드에 할당해야 합니다.  
즉, 다음과 같은 공식을 사용해야 합니다:  
```
v` = S * d
```
여기서:  
* `v'`는 새 속도 벡터입니다,  
* `S`는 속도 상수 `SPEED`입니다,  
* `d`는 방향 벡터입니다.

방향 벡터는 원하는 방식으로 정의할 수 있지만, 다음 제약 조건을 만족해야 합니다:
* `x`와 `y` 좌표는 각각 `1`, `0`, 또는 `-1`이어야 합니다.
* 벡터는 무작위로 생성되어야 하며, 메서드를 여러 번 연속 호출하더라도  
  동일한 방향이 생성될 확률이 낮아야 합니다.

무작위 방향 벡터를 생성하는 데 다음 함수가 유용할 수 있습니다:

```c++
bool    generateBool(float prob = 0.5f);
int     generateInt(int min, int max);
float   generateFloat(float min, float max);
Point2D generatePoint(const Rectangle& boundingBox);
```

이 함수들에 대한 자세한 설명은 `utils.hpp` 파일에 정의와 함께 제공된  
문서에서 확인할 수 있습니다.

`Enemy` 클래스 작업을 계속 진행하면서,  
우선 `getTexture` 메서드를 구현하는 간단한 부분을 살펴보겠습니다.  
이 메서드는 적 객체를 위한 새로운 특별 텍스처를 반환해야 합니다.  
이 텍스처의 ID는 `GameTextureID::BLACKHOLE`입니다.

더 어려운 부분은 충돌 동작을 구현하는 것입니다.  
적 객체가 플레이어 객체와 충돌하면, 플레이어 객체는 비활성화(inactive) 상태가 되어야 합니다.  
이는 플레이어 객체의 상태를 `DESTROYED`로 설정하여 구현할 수 있습니다.  
하지만 적 객체는 플레이어 객체의 `setStatus` 메서드에 직접 접근할 수 없습니다.  
원하는 동작을 구현하려면 사실 `EnemyObject` 클래스의 메서드가 아닌  
`PlayerObject` 클래스의 `onCollision` 메서드를 수정해야 합니다!

<div class="hint">

`PlayerObject::onCollision` 메서드를 구현할 때,  
충돌이 적 객체와 발생했는지, 아니면 다른 종류의 객체와 발생했는지를  
확인하는 것을 기억하세요!

</div>

소비 가능한 객체의 관점에서 보면,  
플레이어와 적 객체 모두 유사하게 동작합니다.  
따라서 소비 가능한 객체의 동작은 수정할 필요가 없습니다.
