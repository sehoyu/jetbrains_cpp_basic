`Scene` 클래스는 또한 _추상 클래스_입니다.  
즉, 순수 가상 메서드를 포함하고 있어서 인스턴스화할 수 없습니다.  
이로 인해 `Scene` 클래스의 다양한 구현을 가질 수 있는 유연성을 제공합니다.

그 중 하나의 구현은 `GameplayStaticScene` 서브클래스에서 제공됩니다  
(파일 `statscene.hpp` 및 `statscene.cpp` 참조).  
이 씬 구현은 정적이라고 불리는데, 이는 미리 정의된 고정된 수의 게임 오브젝트만 포함하기 때문입니다:  
한 개의 플레이어 오브젝트, 한 개의 소모품 오브젝트, 그리고 한 개의 적 오브젝트로 구성됩니다.  
이 오브젝트에 대해서는 이 모듈 후반부에서 다룰 예정입니다.  
이 방식은 이전 버전의 게임 구현에서 배운 링크드 리스트를 사용하여 동적으로 오브젝트를 생성하는 방식에 비해  
확실히 다운그레이드처럼 보일 수 있습니다.  
하지만 걱정하지 마세요. 이 기능은 곧 복구될 것입니다.  
그러나 지금은 정적 씬으로 작업해보겠습니다.

`Scene` 클래스는 매우 중요하고 게임 오브젝트 관리 및 씬을 그리는 작업처럼  
많은 작업을 처리하지만, 게임 자체를 관리하는 책임은 없습니다.  
이 책임은 다른 클래스인 `GameEngine` 클래스가 맡습니다  
(파일 `engine.hpp` 및 `engine.cpp` 참조).  
특히 이 클래스는 애플리케이션 창과 현재 활성화된 씬을 제어합니다.

<div class="hint">

`GameEngine` 클래스의 또 다른 중요한 책임은 씬 전환 기능을 수행하는 것입니다.  
현재로서는 단일 씬(게임 플레이 씬)만 있을 예정이므로  
이 기능이 우리에게 중요하지 않을 수 있습니다.  
그러나 이 강의의 후반부에서 메인 메뉴를 관리하는 씬과 같은  
다른 유형의 씬을 구현하기 시작하면서 유용하게 될 것입니다.

</div>

`GameEngine` 클래스의 가장 중요한 메서드는 `run()` 메서드입니다.  
이 메서드는 게임의 메인 루프를 구현합니다.  
본질적으로, 우리의 게임 애플리케이션의 진입점은 단지  
`GameEngine` 객체를 생성하고 `run()` 메서드를 호출하는 것입니다 (파일 `main.cpp` 참조).

`GameEngine` 객체를 생성하는 것에 대해 말하자면,  
`GameEngine` 클래스는 지금까지 본 다른 클래스와 비교했을 때  
특별한 특징을 가지고 있습니다.  
게임 인스턴스마다 단 하나의 독특한 `GameEngine` 객체 인스턴스만 존재할 수 있도록 합니다.  
C++의 또 다른 기능인 `static` 한정자를 이용하여 코드를 통해 이를 표현할 수 있습니다.

우선, `GameEngine` 클래스에는 다른 메서드와는 차별화된 한 가지 메서드가 있다는 것에 주목하세요:  
`create()` 메서드입니다. 이 메서드는 앞에 `static` 한정자가 붙어 있습니다.  
`static` 한정자는 클래스 멤버(필드나 메서드) 앞에 적용되면  
해당 멤버를 _정적_ 멤버로 만듭니다.

정적 멤버는 객체와 연관되지 않고, 클래스 자체와 연관됩니다.  
즉, 정적 멤버에 접근하려면 클래스의 인스턴스가 필요하지 않습니다.  
대신 정적 멤버는 클래스 이름을 통해 접근합니다:

```c++
// 정적 메서드 `create`를 호출하여 엔진 인스턴스를 얻음
GameEngine* engine = GameEngine::create();
```

정적 멤버는 특정 메서드나 데이터 필드를 클래스 자체와 연결하는  
편리한 방법을 제공합니다.  
예를 들어, 위에서 본 `create` 메서드는 게임 엔진 객체를 인스턴스화할  
능력을 제공합니다.

함수 내부에 선언된 지역 변수의 선언에 `static` 한정자를 적용하면  
그 의미는 달라집니다.  
_정적 변수_를 생성할 수 있으며, 이 변수는 함수 호출 간 값을 유지합니다.  
이러한 변수는 실제로 프로그램의 정적 메모리 영역에 저장되며,  
함수의 다른 지역 변수가 위치하는 스택 메모리 영역에 저장되지 않습니다  
(여기에서 _static_이라는 이름이 유래합니다).

```c++
int foo() {
    static int x = 0;
    return ++x;
}

// 1 출력
std::cout << foo() << std::endl;
// 2 출력
std::cout << foo() << std::endl;
```

`static` 한정자를 사용하면, 게임 실행당 한 번만 엔진이 생성되도록 보장할 수 있습니다.  
이를 달성하려면, `GameEngine::create` 메서드 내부에  
정적 `GameEngine` 변수를 선언하고 해당 변수에 대한 포인터를 반환하면 충분합니다.

<div class="hint">

이 경우 주소 탈출 오류(address escape error)가 발생하지 않음을 주목하세요.  
`static` 변수는 정적 메모리 영역에 위치하므로,  
프로그램 실행 시간 전체 동안 유지됩니다.  
따라서 이 변수의 주소를 함수에서 반환해도 안전합니다.

</div>

위에서 설명한 대로 `create` 메서드를 구현하세요.  
올바르게 구현하면 리팩토링된 게임 애플리케이션을 실행할 수 있으며  
화면에서 행성과 별 오브젝트를 확인할 수 있을 것입니다.
