C++에서, 배열 타입의 변수는 내부적으로 배열의 첫 번째 요소를 가리키는 포인터로 표현됩니다.  
따라서, 예를 들어 배열 변수 `int a[]`가 주어졌을 때, 다음 표현들은 같은 의미를 가집니다:

| 표현                           | 배열 표기법          | 포인터 표기법              |
|--------------------------------|---------------------|---------------------------|
| 배열에 대한 포인터 초기화       | `int* p = &a[0]`    | `int* p = a`              |
| 인덱스를 통한 요소 접근         | `a[n]`              | `*(a + n)`                |

이로 인해, 배열 인수를 기대하는 함수는 두 개의 인수를 가진 함수로 작성될 수 있습니다:  
* 배열의 첫 번째 요소에 대한 포인터, 그리고  
* 배열의 크기.

```c++
void f(int* p, size_t size);
```

C++ 코드에서는 또 다른 방식의 인코딩을 자주 볼 수 있습니다.  
동일한 함수는 두 가지 서로 다른 유형의 인수를 가진 함수로 인코딩될 수 있습니다:  
* 배열의 첫 번째 요소에 대한 포인터, 그리고  
* __배열의 마지막 요소를 넘어서는 첫 번째 요소에 대한 포인터__.

```c++
void g(int* begin, int* end);
```

다음과 같이 정의된 배열 `a`가 주어졌다고 가정합니다:

```c++
const size_t SIZE = 10;
int a[SIZE];
```

이때 위 함수들은 아래와 같이 호출해야 합니다:

```c++
f(a, SIZE);
```

그리고

```c++
g(a, a + SIZE);
```

두 번째 방식의 인코딩에서는 배열의 모든 요소를 순회하는 반복문을 다음과 같이 작성할 수 있습니다:

```c++
for (int* p = begin; p < end; ++p) {
    std::cout << *p << "\n";
}
```

<div class="hint">

위의 코드가 한쪽 포인터가 null 포인터일 경우를 처리하지 않는다는 점을 유의하세요.

</div>

이 관용구(idiom)는 C++ 코드에서 매우 자주 사용됩니다.  
따라서, 이것이 어떻게 작동하는지를 이해하는 것이 매우 중요합니다.  
이를 이해했는지 확인하기 위해 두 함수를 구현해 볼 수 있습니다:  
* `sum_size`, 그리고  
* `sum_ptrs`.

이 두 함수는 모두 동일한 작업을 수행해야 합니다:  
배열의 모든 요소들의 합을 계산하는 함수.  
다만, 첫 번째 함수는 배열의 시작 포인터와 배열 크기를 인수로 받고,  
두 번째 함수는 배열의 시작과 끝에 대한 포인터를 인수로 받습니다.  
두 함수 모두 받은 포인터들이 유효하고 null이 아님을 가정할 수 있습니다.
