## 메모리 관련 버그와 이를 피하는 방법

C/C++에서 수동 메모리 관리는 프로그래머에게 큰 권한을 제공하지만, 이 권한에는 큰 책임이 따릅니다!  
메모리 관리에서의 실수는 충돌, 데이터 손상 또는 보안 취약점과 같은 다양한 문제를 초래할 수 있습니다.  
이 단계에서는 흔히 발생하는 몇 가지 메모리 관련 버그에 대한 예를 제공하고,  
CLion IDE와 뛰어난 서드파티 도구의 도움으로 이러한 성가신 버그들을 발견하고 해결할 수 있는 방법을 보여드리겠습니다.

### 주소 탈출(Address Escape)

여러분은 이미 _주소 탈출(address escape)_ 메모리 오류와 친숙할 것입니다.  
이 오류는 함수의 로컬 범위 내 메모리 위치를 가리키는 포인터가  
함수 밖으로 "탈출"하여 함수가 반환된 후에도 사용되는 경우에 발생합니다.  
이전 단계 중 하나에서 살펴본 예를 떠올려 보세요:

```c++
int* escape() {
    int a = 42;
    return &a;
}

void escapeCaller() {
    int* p = escape();
    std::cout << (void*) p << "\n";
}
```

### 범위를 벗어난 접근(Out-of-Bounds Access)

우리가 이미 언급한 또 다른 메모리 관련 버그는 _범위를 벗어난 접근(out-of-bounds access)_입니다.  
간단히 말하면, 이는 할당된 범위를 벗어나 메모리에 접근하는 경우를 말합니다.

```c++
char* a = (char*) malloc(8);
// 유효한 접근
*(a + 2) = 1;
// 범위를 벗어난 접근
*(a + 9) = 2;
// 이것도 범위를 벗어난 접근
*(a - 1) = 3;
```

### 메모리 누수(Memory Leak)

동적으로 할당된 메모리 블록을 해제하려면 해당 메모리를 가리키는 포인터가 필요합니다.  
만약 이 포인터를 잃어버린다면, 메모리를 해제할 수 없게 됩니다.  
이러한 프로그래밍 오류를 _메모리 누수(memory leak)_라고 합니다.

메모리 누수 예제를 보려면 다음 함수를 살펴보세요.

```c++
void leak() {
    char* p = (char*) malloc(8);
    std::cout << (void*) p << "\n";
}
```

이 함수는 8바이트의 메모리 블록을 할당한 후 종료됩니다.  
하지만 할당된 메모리를 가리키는 포인터는 호출자에게 반환되거나  
전역 변수에 저장되지 않습니다.  
실제로 함수가 완료되면 이 포인터는 사라지게 되고, 이는 메모리 누수를 초래합니다.  
즉, 할당된 메모리 블록은 다시 회수할 수 없게 됩니다.

### 해제 이후 사용(Use After Free)

한 번 메모리 블록이 해제되면,  
이를 참조하던 포인터를 역참조하거나  
다른 방식으로 해제된 메모리에 접근하려는 시도는 정의되지 않은 동작을 유발합니다.  
실제로 이는 프로그램이 충돌하거나 데이터 손상이 발생하는 결과를 초래할 수 있습니다.

```c++
char* p = (char*) malloc(8);
free(p);
// 정의되지 않은 동작
*p = 7;
```

### 이중 해제(Double Free)

마찬가지로, 이미 해제된 메모리를 다시 한 번 해제하려는 시도 역시 문제를 발생시킬 수 있습니다:

```c++
char* q = (char*) malloc(8);
free(q);
// 정의되지 않은 동작
free(q);
```

### 메모리 버그 탐지

물론 프로그래머들은 프로그램 내 메모리 오류를 탐지하는  
훌륭한 도구들을 이미 개발해 왔으며, CLion IDE는 이러한 도구와의  
편리한 통합 기능을 제공합니다.

메모리 관련 버그를 탐지하는 도구들은 프로그램의 _런타임 모니터링_을 수행합니다.  
이 도구들은 프로그램에 추가적인 검사 코드를 삽입하여 메모리 관리 규칙을  
위반하지 않도록 보장합니다. 이러한 검사는 프로그램 실행 중에 수행됩니다.  
따라서 메모리 오류 탐지기를 사용하면 약간의 성능 저하가 있을 수 있습니다.  
이로 인해 이러한 도구들은 보통 기본값으로 활성화되어 있지 않으며,  
오직 디버깅이나 프로그램 테스트 중에만 사용됩니다.

메모리 오류를 탐지하는 데 도움이 되는 특히 인기 있는 두 가지 도구가 있습니다:  
- [AddressSanitizer](https://www.jetbrains.com/help/clion/google-sanitizers.html#AsanChapter)  
- [Valgrind Memcheck](https://www.jetbrains.com/help/clion/memory-profiling-with-valgrind.html)  

위 링크를 클릭하여 CLion 문서를 참고하면서 IDE에서 이 도구들 중 하나(또는 둘 모두)를 설정해 보세요.  
그런 다음, 이 단계에 첨부된 프로그램에서 메모리 오류 탐지기를 실행해 보세요.  
이 프로그램은 여기에서 언급된 모든 유형의 메모리 관련 버그를 포함하고 있습니다.  
메모리 오류 탐지기가 이러한 모든 버그를 찾아낼 것입니다!
