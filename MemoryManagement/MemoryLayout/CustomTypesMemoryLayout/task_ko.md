다음으로, 사용자 정의 타입: 열거형(enum)과 구조체(struct)의 메모리 레이아웃에 대해 논의해 보겠습니다.

열거형(enum)은 내부적으로 단순히 정수로 표현됩니다.  
다음은 enum 타입의 선언 예제입니다:

```c++
enum Color {
    RED, GREEN, BLUE
};
```

다음 코드를 실행하여 `Color` 타입과 `int` 타입이 동일한 메모리 크기를 차지하는지 확인해 보세요:

```c++
std::cout << sizeof(int) << "\n";
std::cout << sizeof(Color) << "\n";
```

구조체(struct)의 메모리 레이아웃은 더 복잡합니다.  
이는 구성 요소의 크기뿐만 아니라 __메모리 정렬(alignment)__도 고려하기 때문입니다.

[메모리 정렬](https://en.cppreference.com/w/cpp/language/object#Alignment)은 특정 타입의 변수들이 할당될 수 있는 연속적인 메모리 주소 간의 바이트 단위를 의미합니다.  
타입의 메모리 정렬은 `alignof` 연산자를 사용하여 조회할 수 있으며, 이는 `size_t` 타입의 값을 반환합니다.  
`int` 나 `double`과 같은 기본적으로 정의된 원시 타입의 경우, 정렬은 일반적으로 타입의 크기와 동일하지만 모든 타입에서 항상 그런 것은 아닙니다.

```c++
std::cout << alignof(int) << "\n";
// 대부분의 현대 머신에서 4가 출력됩니다.
std::cout << alignof(double) << "\n";
// 대부분의 현대 머신에서 8이 출력됩니다.
```

<div class="hint">

C++ 언어에 [메모리 정렬](https://en.wikipedia.org/wiki/Data_structure_alignment) 개념을 도입한 동기는 프로세서가 메모리를 운영하는 저수준 작동 방식에서 비롯됩니다.  
가령, 메모리에 4바이트 데이터를 읽거나 쓰는 경우,  
프로세서는 접근한 메모리 주소가 4의 배수에 정렬되어 있는(_aligned_) 상태이길 기대합니다.  
그렇지 않으면 이런 메모리 접근은 _unaligned_라 불립니다.  
구체적인 프로세서 아키텍처에 따라 정렬되지 않은 메모리 접근은 느리거나 오류를 발생시킬 수 있습니다.

</div>

사용자 정의 구조체의 메모리 레이아웃은 각 필드의 정렬을 고려합니다.  
필요하다면 컴파일러는 구조체의 모든 멤버를 적절하게 정렬하기 위해 _패딩(padding)_을 삽입합니다.  
다음 구조체를 살펴보세요.

```c++
struct Foo {
    int x;
    char a;
    char b;
    int y;
};
```

`int`가 4바이트를 차지한다고 가정할 때, 이 구조체의 크기는 12바이트이고 정렬 크기는 4바이트입니다.  
이는 컴파일러가 `b`와 `y` 필드 사이에 2바이트의 패딩을 추가하기 때문입니다.

```c++
std::cout << sizeof(Foo) << "\n";
// 12가 출력됩니다.
std::cout << alignof(Foo) << "\n";
// 4가 출력됩니다.
```

즉, `Foo` 구조체의 내부 메모리 표현은 다음 구조체의 메모리 표현과 동일합니다:

```c++
struct FooLayout {
    int x;
    char a;
    char b;
    char _padding[2];
    int y;
};
```

이를 크기와 정렬을 출력하여 확인할 수 있습니다.

```c++
// 12가 출력됩니다.
std::cout << sizeof(FooLayout) << "\n";
// 4가 출력됩니다.
std::cout << alignof(FooLayout) << "\n";
```

정렬 요구사항으로 인해, 동일한 필드를 가지지만 선언 순서가 다른 두 구조체는 크기가 다를 수 있습니다.

예를 들어, `Bar`라는 이름의 다음 구조체를 살펴보세요:

```c++
struct Bar {
    int x;
    char a;
    int y;
    char b;
};
```

`Foo` 구조체와 달리, 이 구조체의 크기는 16바이트입니다.

```c++
// 16이 출력됩니다.
std::cout << sizeof(Bar) << "\n";
// 4가 출력됩니다.
std::cout << alignof(Bar) << "\n";
```

구조체에 대한 포인터를 사용하면 `->` 멤버 접근 연산자를 이용하여 구조체의 필드에 접근할 수 있습니다.  
가령, `p`가 `Foo`의 포인터(`Foo*`)라고 하면, 다음과 같은 표현식은 동일한 의미를 가집니다:

| 연산자           | 표기           |
|-------------------|---------------|
| 구조체 멤버 접근 | `(*p).x`      |
| 포인터 멤버 접근 | `p->x`        |

구조체의 필드에 접근하기 위해, 컴파일러는 각 필드의 오프셋(offset)을 계산합니다.  
예를 들어, `Bar` 구조체에서 필드의 오프셋은 다음과 같습니다:
* `x` --- 0바이트,
* `a` --- 4바이트,
* `y` --- 8바이트,
* `b` --- 12바이트.

구조체의 할당된 메모리에서 해당 주소를 출력하면 이러한 필드 오프셋을 확인할 수 있습니다.

```c++
Bar bar;
std::cout << (void*) &bar << "\n";
std::cout << (void*) &(bar.x) << "\n";
std::cout << (void*) &(bar.a) << "\n";
std::cout << (void*) &(bar.y) << "\n";
std::cout << (void*) &(bar.b) << "\n";
```

`(void*)` 부분에 주목하세요.  
이는 _C 스타일 형변환 연산자_입니다.  
이 강좌의 뒷부분에서 이에 대해 다룰 예정입니다.  
여기서는 변수 타입이 `char*`인 값을 문자열이 아닌 메모리 주소로 출력하기 위해 필요합니다.

구조체의 필드에 접근하려면 해당 필드의 오프셋을 구조체 메모리 주소에 더하기만 하면 됩니다.  
필드의 오프셋을 얻는 신뢰할 수 있고 플랫폼에 독립적인 방법은 `offsetof` 연산자를 사용하는 것입니다:

```c++
// p는 bar가 점유한 메모리의 시작 위치를 가리킵니다.
char* p = (char*)&bar;
// 이제 p는 bar의 y 필드가 점유한 메모리를 가리킵니다.
p += offsetof(struct Bar, y);
// 이를 int 포인터로 변환합니다.
int* pi = (int*) p;
// 이제 y 필드에 값을 할당할 수 있습니다.
* pi = 42;
// 42가 출력됩니다.
std::cout << bar.y << "\n";
