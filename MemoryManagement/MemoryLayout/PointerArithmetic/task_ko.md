메모리는 주소 지정이 가능한 메모리 셀의 연속이라는 것을 기억하세요.  
그렇다면 특정 메모리 셀의 주소를 받으면,  
이 연속체에서 "다음 단계로 이동"하여 "다음" 메모리 셀을 볼 수 있다는 뜻일까요?  
또는, 5단계 정도의 거리에 있는 메모리 셀로 점프할 수 있을까요?  
혹은 두 메모리 주소를 보고 그들 사이의 거리를 계산할 수 있을까요?

대답은 "예"입니다. 그러나 몇 가지 미묘한 사항이 있습니다.

`char` 타입의 배열을 생각해봅시다.

```c++
const int SIZE = 8;
char a[SIZE] = "abcdefgh";
```

<div class="hint">

`char`는 ASCII 문자를 인코딩하기 위해 사용되는 타입임을 기억하세요.  
이 타입은 메모리에서 주소 지정이 가능한 최소 단위인 1 바이트 크기를 가지는 것이 보장됩니다.  
따라서, `char*` 타입을 사용하여 메모리 셀의 주소를 저장하는 것이  
종종 편리합니다.

</div>

이제 이 배열의 첫 번째 요소를 가리키는 포인터를 정의해봅시다.

```c++
char* p0 = &a[0];
```

이제 `p0`는 배열의 첫 번째 요소를 가리킵니다.

```c++
std::cout << *p0 << "\n";
// a를 출력합니다.
```

다음 요소를 살펴보기 위해 한 단계 더 이동하려면,  
포인터 `p0`에 정수 값 `1`을 더하면 됩니다.

```c++
char* p1 = p0 + 1;
```

이제 `p1`은 배열의 두 번째 요소를 가리킵니다.

```c++
std::cout << *p1 << "\n";
// b를 출력합니다.
```

새로 계산된 포인터를 변수를 사용하여 저장할 필요는 없습니다.  
포인터를 직접 역참조할 수도 있습니다.

```c++
std::cout << *(p0 + 1) << "\n";
// b를 출력합니다.
```

포인터에 대해 증가 및 감소 연산도 사용할 수 있습니다.

```c++
char* p = p0;
++p;
std::cout << *p << "\n";
// b를 출력합니다.
```

마찬가지로 배열의 임의의 요소를 가리키는 포인터를 얻는 것도 가능합니다.

```c++
std::cout << *(p0 + 5) << "\n";
// f를 출력합니다.
```

그러나 배열 `a`가 차지하는 메모리 영역 내에서만 점프할 수 있습니다!  
배열이 차지한 메모리 영역 외부의 메모리 셀에 접근하면 정의되지 않은 동작이 발생합니다!  
예를 들어 다음 코드 스니펫은 잘못된 코드입니다.

```c++
std::cout << *(p0 + 20) << "\n";
```

이 코드가 프로그램을 즉시 중단시키지 않을 수도 있습니다.  
하지만 – 이 경우가 더 나쁠 수도 있는데 – 프로그램의 동작은 정의되지 않습니다:  
임의의 값을 출력하거나, 아무것도 출력하지 않거나,  
전혀 다른 동작을 수행할 수도 있습니다.

<div class="hint">

`정의되지 않은 동작`이라는 용어를 이전에 본 적이 있습니다.  
언젠가 이 용어가 의미하는 바와 실제 결과에 대해  
더 자세히 설명할 것을 약속합니다.  
지금은 프로그램에서 `정의되지 않은 동작`을 피해야 한다는 것만 기억하세요.

</div>

포인터에서 정수 값을 빼는 것도 유효한 연산입니다:  
단, 포인터가 배열의 메모리 영역 내에 머물러야 한다는 점을 기억하세요.

```c++
char* q = p1 - 1;
std::cout << *q << "\n";
// a를 출력합니다.
```

또한 두 포인터 간의 차이를 계산하여  
메모리 셀 개수로 거리 값을 얻는 것도 가능합니다.

```c++
char* p8 = p0 + 8;
int diff = p8 - p0;
std::cout << diff << "\n";
// 8을 출력합니다.
```

포인터에 대한 산술 연산의 동작은 실제로 포인터 타입에 따라 다릅니다.  
즉, 포인터가 가리키는 타입의 크기에 따라 달라집니다.

예를 들어, `char` 대신 `int` 타입의 배열을 생각해봅시다.

```c++
int b[SIZE] = { 0, 1, 2, 3, 4, 5, 6, 7 };
```

배열의 시작을 가리키는 포인터를 가져오고 이를 `1`씩 이동하면,   
두 번째 메모리 셀이 아니라 배열의 두 번째 요소를 가리키는 포인터를 얻게 됩니다.

```c++
int* r = &b[0];
++r;
std::cout << *r << "\n";
// 1을 출력합니다.
```

`int`는 일반적으로 `4` 바이트를 차지합니다.  
이 가정 하에 변수 `r`은 `4`번째 메모리 셀을 가리킵니다.

일반적으로 타입 `T*`의 포인터가 있을 때, 표현식 `p + n`은  
메모리 셀 단위로 측정할 경우 `p + n * sizeof(T)`와 동일합니다.  
즉, 포인터를 타입의 크기를 단위로 하는 측정 도구로 볼 수 있습니다.

결론적으로, 위에서 언급한 모든 포인터 연산은 **포인터 산술**로 알려져 있습니다.  
여기에는 다음과 같은 연산이 포함됩니다:

* 포인터에 정수 오프셋 더하기: `p + n`;
* 포인터에서 정수 오프셋 빼기: `p - n`;
* 포인터 증가 및 감소: `p++`, `--p` 등;
* 포인터 간 거리 계산: `p - q`.

위에서 `p`와 `q`는 포인터이며, `n`은 정수입니다.

포인터 산술 연산자는 동일한 배열에 속하는 포인터에만 적용 가능합니다.  
즉, 메모리 영역 `p` .. `p + size` (포함된 범위) 내에서만 사용할 수 있습니다.  
여기서:  
* `p`는 배열의 시작을 가리키는 포인터이며,  
* `size`는 배열의 크기입니다.

이러한 산술 연산 외에도 포인터에 대해 비교 연산자를 적용할 수 있습니다:

* 포인터의 (비)동일성 확인: `p == q`, `p != q`;
* 포인터 비교: `p <= q`, `p > q` 등.

모든 두 개의 포인터를 (비)동일성으로 확인할 수 있지만,  
동일한 메모리 영역(즉, 동일한 배열에 의해 점유된)의 포인터만  
`<=`와 같은 연산자를 사용하여 비교할 수 있습니다.
