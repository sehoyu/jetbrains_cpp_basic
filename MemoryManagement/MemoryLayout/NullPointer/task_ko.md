C++에서는 특별한 **null pointer** 값인 `nullptr`가 있습니다.  
null pointer는 실제 메모리 위치를 가리키지 않습니다.  
null pointer를 참조 해제하는 것은 *정의되지 않은 동작*을 유발하며,  
대부분 런타임 오류를 일으키고 프로그램을 중단시킬 것입니다.  

<div class="hint">

*정의되지 않은 동작*의 개념은 이 강좌의 후반부에서 다시 다룰 예정입니다.  
지금은 이것이 정말로 **좋지 않은** 일이므로 반드시 피해야 한다는 점만 기억하세요.  

</div>

```c++
int* p = nullptr;
// 0 출력
std::cout << p << "\n";
// 프로그램이 중단됨
std::cout << *p << "\n";
```

null pointer는 참조 해제가 불가능하지만,  
그렇다고 해서 완전히 쓸모없는 것은 아닙니다.  
예를 들어, null pointer는 이후 실제 주소로 재할당될  
포인터를 초기화하는 데 사용할 수 있습니다.  
하지만, 포인터가 실제 주소로 재할당되기 전에는  
이를 참조 해제하지 않도록 주의해야 합니다.  

<div class="hint">

`nullptr`는 C++에 고유한 키워드입니다.  
C 언어에서는 이에 상응하는 상수로 `NULL`이 있습니다.  

</div>

포인터를 받는 함수가 있고, 해당 포인터가 null이 아님을  
보장할 수 없는 경우, 포인터를 참조 해제하기 전에 이를  
검증할 수 있습니다.  

```c++
void f(int* p) {
    if (p != nullptr) {
        int v = *p;
        // 실제 작업 수행    
    }
}
```

C++에서는 `nullptr`이 정수값이 필요한 상황에서  
정수 값 `0`으로 평가될 것을 보장합니다.  
따라서, 위 코드 조각은 다음과 동일합니다:  

```c++
void f(int* p) {
    if (p) {
        int v = *p;
        // 실제 작업 수행    
    }
}
