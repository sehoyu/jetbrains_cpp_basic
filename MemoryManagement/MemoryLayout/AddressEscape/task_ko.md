스택 메모리 관리 정책으로 인해,  
지역 변수의 주소는 호출 스택에서 더 하위에 있는 다른 함수로 전달될 수 있지만,  
절대로 함수 호출자(caller)로 반환될 수는 없습니다.  
그 이유는 함수가 반환될 때 해당 함수의 지역 변수가 점유하던 메모리가 해제되기 때문입니다.  

구체적인 예를 들기 위해, 다음 코드 조각은 완전히 유효합니다.  
`caller` 함수 내부에 정의된 지역 변수의 주소가 `callee` 함수에 전달됩니다.

```c++
void callee(int* p) {
    std::cout << p << " -> " << *p << "\n";
}

void caller() {
    int a = 42;
    callee(&a);
}
```

실제로, `callee`가 `caller`로부터 호출되었기 때문에,  
`caller`와 관련된 스택 프레임은 `callee`의 스택 프레임이  
스택에서 제거되기 전에 제거될 수 없습니다.  
따라서, 변수 `a`가 점유하는 메모리는 `callee`가 실행되는 동안  
내내 할당된 상태로 남게 됩니다.  

반대로, 다음 코드 조각은 오류가 발생합니다.

```c++
int* escape() {
    int a = 42;
    return &a;
}

void escapeCaller() {
    int* p = escape();
    std::cout << p << " -> " << *p << "\n";
}
```

이 경우, 변수 `a`의 주소가 함수 밖으로 _탈출_ 하게 됩니다.  
이 주소는 호출자에게 전달되며, 그 주소에 접근하려고 할 때에는  
이미 `escape` 함수의 스택 프레임이 스택에서 제거된 상태입니다.  
따라서 변수 `a`가 점유하던 메모리는 해제된 상태가 됩니다.  
위 코드를 컴파일하고 실행하려고 하면, 프로그램이 비정상적으로  
종료되거나 정크 데이터를 출력할 가능성이 큽니다.  

유사한 오류는 지역 변수의 주소를 전역 변수에 저장할 때도 발생합니다.

```c++
int* q = nullptr;

void escapeGlobal() {
    int a = 42;
    q = &a;    
}

void escapeGlobalCaller() {
    escapeGlobal();
    std::cout << q << " -> " << *q << "\n";
}
