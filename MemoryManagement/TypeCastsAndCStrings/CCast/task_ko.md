다른 유형의 변수로 한 유형의 변수를 처리해야 하는 상황은 매우 많습니다.  
예를 들어, 표준 라이브러리의 `sqrt` 함수에 대해 생각해 봅시다.  
이 함수는 제곱근을 계산하며 `double` 유형의 인수를 기대합니다.  
그러나 만약 우리가 정수, 예를 들어 `4`의 제곱근을 계산하고 싶다면 어떻게 해야 할까요?  
만약 `sqrt` 함수에 `int` 변수를 전달하려고 하면,  
그 변수는 _암시적으로 변환되어_ `double`로 처리됩니다.

```c++
int x = 4;
// 2.0이 출력됩니다.
std::cout << sqrt(x) << "\n";
```

C와 C++ 언어는 암시적 유형 변환이 언제, 그리고 어떻게 수행될 수 있는지에 대해  
광범위하고 복잡한(그리고 약간 다른) [규칙](https://en.cppreference.com/w/cpp/language/implicit_conversion)을 가지고 있습니다.  
이 규칙에는 숫자 유형 간의 변환(예: `int` 또는 `double`)이 포함됩니다.  
중요하게도 정보 손실을 초래할 수 있는 변환도 허용됩니다. 
예를 들어, `double` 변수는 `int`로 변환될 수 있습니다.

```c++
double pi = 3.14;
int a = pi;
// 2가 출력됩니다.
std::cout << a << "\n";
```

그러나 모든 합리적인 유형 캐스팅이 암시적으로 수행되는 것은 아닙니다.  
이런 경우 _C 스타일 유형 변환 연산자_를 사용할 수 있습니다.  
우리는 이미 `malloc`을 다룰 때 이를 본 적이 있습니다.  
`malloc`은 단순히 연속된 메모리 영역을 할당하며,  
특정 유형이 지정되지 않았습니다. 따라서 이 함수는 `void*` 유형의 포인터를 반환합니다.  
만약 정수 배열을 할당하려는 경우, 이 포인터는 명시적으로 `int*`로 캐스팅될 수 있습니다:

```c++
void* q = malloc(2 * sizeof(int));
int* qi = (int*) q;
qi[0] = 42;
```

C 스타일 유형 변환 연산자는 매우 관대하며  
어떠한 안전성 검사도 수행하지 않는다는 점에 유의하세요.  
이를 잘못 사용하면 정의되지 않은 동작을 초래할 수 있습니다.  
예를 들어, 메모리 2바이트만 할당하고  
2개의 `int`에 충분한 메모리를 할당한 것처럼 가정하는 다음 프로그램을 생각해 봅시다.  
그런 다음 할당된 메모리에 `int` 값을 작성하려고 시도합니다  
(가정: `sizeof(int) == 4`).  
이 쓰기 작업은 할당된 범위를 벗어난 메모리에 영향을 미칠 수 있습니다.  
이는 예상치 못한 결과를 초래할 수 있습니다.

```c++
void* r = malloc(2);
int* ri = (int*) r;
// 정의되지 않은 동작 !!!
ri[0] = 42;
```

이러한 이유로, C 스타일 유형 변환은 주의해서 사용해야 합니다.  
C 스타일 캐스트에서 허용되는 유형 변환 규칙의 전체 집합은  
[여기](https://en.cppreference.com/w/c/language/cast)에서 확인할 수 있습니다.
