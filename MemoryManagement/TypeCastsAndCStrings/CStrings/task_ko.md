우리는 이미 텍스트 데이터를 표현하는 데 사용되는 C++ 타입인 `std::string` 타입을 살펴보았습니다.  
짐작하셨겠지만, C에는 텍스트 데이터를 처리하는 고유한 방법이 있습니다.  
C에서 문자열은 `char` 타입의 포인터로 표현됩니다:

```c++
const char* greetings = "Hello World!";
std::cout << greetings << "\n";
```

`char*` 타입 변수는 문자열의 길이에 대한 정보를 저장하지 않는다는 점에 유의하세요.  
C 언어 규칙에 따라, 문자열은 특별 문자 `\0`로 종료되어야 하며, 이는 문자열의 끝을 나타냅니다.  
따라서 문자열의 길이는 `\0` 문자가 발견될 때까지 문자를 반복적으로 검사함으로써 계산됩니다.  
이를 위해 C 표준 라이브러리는 정확한 목적에 맞는 함수를 제공합니다:

```c++
// 12를 출력합니다
std::cout << strlen(greetings) << "\n";
```

C 스타일 문자열은 길이를 명시적으로 저장하지 않기 때문에,  
버퍼 오버플로 오류를 방지하기 위해 프로그래머가 이를 직접 관리해야 합니다.  
예를 들어, 입력에서 C 스타일 문자열을 읽을 때에는 문자열의 길이를 고려하고,  
문자열 끝을 넘어 메모리가 덮어쓰이지 않도록 하는 특별 함수 `getline`을 사용하는 것이 중요합니다:

```c++
const size_t size = 20;
char name[size];
std::cin.getline(name, size);
std::cout << name << "\n";
```

<div class="hint">

내부적으로 `char[]` 배열은 메모리에서 포인터 `char*`와 동일한 방식으로 표현된다는 점을 기억하세요.  
따라서 위 코드에서 변수 `name`도 C 스타일 문자열로 간주됩니다.

</div>

C 스타일 문자열을 C++ 스타일 문자열로 변환하거나 그 반대도 가능합니다:

```c++
std::string cppString = std::string(greetings);
const char* cString = cppString.c_str();  
```

<div class="hint">

C나 C++ 언어 모두에서, 프로그램 텍스트 내에서 정의된 문자열(위의 "Hello World!"와 같은)은  
[_문자열 리터럴_](https://en.cppreference.com/w/cpp/language/string_literal)  
(또는 _정적 문자열_)이라고도 불립니다.

문자열 리터럴은 정적 메모리의 읽기 전용 불변 영역에 저장됩니다.  
그러나 C 언어는 문자열 리터럴 값을 non-const 메모리로의 포인터에 저장하는 것을 허용합니다:

```c++
// 이는 유효한 C 코드입니다
char* str = "I am immutable!";
```

정적 문자열을 수정하려는 시도는 undefined behavior(정의되지 않은 동작)를 초래합니다.

```c++
// 이 코드는 정의되지 않은 동작을 유발합니다!
str[5] = " ";
str[6] = " ";
```

C++ 언어는 정적 문자열에 대한 포인터를 `const char*`로 선언할 것을 요구합니다.  
그렇지 않으면, 위 코드와 같은 경우 컴파일 오류로 인해 코드가 거부됩니다.

C에서 같은 요구를 하지 않는 이유는 순전히 역사적인 이유 때문입니다:  
C 언어 초기에는 `const` 키워드가 없었으며, 따라서 constantness 규칙을 검토하지 않았습니다.  

</div>
