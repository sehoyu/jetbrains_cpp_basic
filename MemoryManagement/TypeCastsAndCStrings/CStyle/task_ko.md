C++ 프로그래밍 커뮤니티에서는 흔히 C 스타일과 C++ 스타일의 프로그래밍에 대해 들을 수 있습니다.  
이 두 가지 스타일의 예제를 다양한 C/C++ 프로젝트에서 찾을 수도 있습니다.  
그렇다면 정확히 이 두 스타일은 무엇일까요?

처음에 C++ 언어는 C 언어의 확장판으로 개발되었습니다.  
현재에도 C로 작성된 대부분의 코드는 C++ 컴파일러에서도 컴파일할 수 있습니다.  
하지만 일반적으로 C는 C++의 엄격한 부분 집합이 아니며,  
이 두 언어 사이에는 많은 [비호환성](https://en.wikipedia.org/wiki/Compatibility_of_C_and_C%2B%2B)이 존재합니다.  
게다가 C와 C++에는 서로 다른 언어 기능, 관용구, 패턴이 많습니다.  
이런 차이점들이 앞서 말한 C 스타일과 C++ 스타일을 형성합니다.

미래에 순수하게 C++ 스타일만 따르더라도 C 스타일 코드를 읽는 방법을 배우는 것이 중요합니다.  
현재 C 언어는 "크로스 플랫폼 어셈블리 언어"로 자리 잡았으며,  
다른 언어들이 서로, 기반 운영체제, 그리고 시스템 수준 소프트웨어와 통신할 때  
자주 사용하는 범용 저수준 언어가 되었습니다.  
많은 C++ 프로젝트 또한 C로 작성된 라이브러리와 통합해야 하므로, 가끔은 C 스타일의 코딩을 사용해야 할 때가 있습니다.

우리의 과정에서 이미 두 스타일의 몇 가지 예제를 보았습니다.

- 지금까지 우리는 입력 및 출력(I/O)을 위해 C++ 스타일 원시 개체인  
  `std::cout`와 `std::cin`을 사용했습니다.  
  C 언어는 I/O를 수행하기 위한 독자적인 API를 가지고 있습니다.  
  이 API에 대해서는 이후에 다시 살펴보겠습니다.

- 첫 번째 모듈에서 우리는 `rand()` 함수를 사용하여 난수를 생성했습니다.  
  사실, 이 함수는 C 표준 라이브러리의 일부입니다.  
  C++은 더 고급스럽고 (따라서 이해하기 더 복잡한) 난수 생성 기능을 제공합니다.  
  이에 대해서는 과정의 뒷부분에서 다룰 예정입니다.

- 우리가 이번 모듈에서 배운 메모리 관리 함수들 대부분  
  (예: `malloc` 및 `free`)은 저수준 C 스타일 원시 개체로 간주됩니다.  
  C++은 다른 대안을 제공하며, 이는 다음 모듈에서 자세히 다룰 예정입니다.  
  C++ 메모리 관리 원시 개체는 우리가 아직 배우지 않은 기능들,  
  예를 들어 _객체 시스템_과 _자원 소유권_의 개념과 밀접하게 연결되어 있습니다.  
  반면, C 스타일 메모리 관리는 "더 근본적인" 방식에 가깝습니다.  
  따라서 더 복잡한 개념으로 넘어가기 전에 이에 대한 정신적 모델을 먼저 구축하는 것이 중요합니다.

이 강의에서는 C와 C++ 스타일로 다르게 표현되는 중요한 두 가지 원시 개체, 즉  
형 변환(type casts)과 문자열(strings)에 대해 학습할 것입니다.
