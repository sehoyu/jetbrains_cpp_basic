우리는 이미 C++가 프로그래머가 포인터를 통해 변수의 주소를 조작할 수 있도록 허용한다는 것을 보았습니다.  
이 언어는 또한 _레퍼런스_라는 또 다른 원시 수단을 제공하여 간접적으로 변수를 조작할 수 있게 합니다.  

레퍼런스는 상수 포인터와 유사합니다:  
레퍼런스는 변수의 주소를 저장하지만 다시 할당할 수 없습니다.  

```c++
int x = 1;
// r은 x에 대한 레퍼런스입니다
int& r = x;
```

보시는 것처럼, 레퍼런스는 `&` 연산자를 참조할 타입에 추가하여 정의됩니다.  

간접 참조 구문(연산자 `*`)을 사용하지 않고, 일반 할당 연산자를 통해서도  
참조된 변수의 값을 변경할 수 있습니다.  

```c++
r = 2;
// 출력: 2
std::cout << r << "\n";
// 출력: 2 (x도 출력)
std::cout << x << "\n";
```

또한 `const` 수정자를 사용하여 읽기 전용(불변) 레퍼런스를 정의할 수도 있습니다:

```c++
const int& cr = x;
// 출력: 2
std::cout << cr << "\n";
// 다음 줄은 컴파일되지 않습니다
cr = 3;
```

포인터와 레퍼런스 간에는 몇 가지 중요한 차이가 있습니다.  

첫째, 포인터 변수와 달리 레퍼런스는 초기화 없이 사용할 수 없습니다:  
레퍼런스를 정의할 때 반드시 어떤 변수에 참조를 할당해야 합니다.  

```c++
// 다음 코드는 컴파일되지 않습니다
int& s;
```

둘째, 레퍼런스는 `nullptr` 값으로 초기화할 수 없습니다.  

```c++
// 이 코드도 컴파일되지 않습니다
int& t = nullptr;
```

셋째, 레퍼런스 변수 자체의 메모리 주소를 얻을 수 없습니다.  
레퍼런스 변수에 주소 연산자를 적용하려고 하면,  
대신 참조된 변수의 주소를 반환합니다.  

```c++
// 다음 두 문장은 항상 동일한 주소를 출력합니다
std::cout << &x << "\n";
std::cout << &r << "\n";
```

위 규칙의 부산물로, 중첩 레퍼런스(예: `int&&`)를 만들 수 없으며  
레퍼런스 배열을 정의할 수도 없습니다.  

다시 말해, 변수에 대한 레퍼런스는 단순히 그 변수의 별칭으로 간주될 수 있습니다.  
이로 인해 컴파일러는 레퍼런스를 처리하는 데 있어 약간의 자유를 얻게 됩니다.  
예를 들어, 컴파일러는 레퍼런스를 저장하기 위한 메모리를  
할당할 의무가 없으며, 가능하면 레퍼런스에 대한 연산을  
참조된 변수에 대한 단순한 접근으로 최적화할 수 있습니다.  

앞서 언급한 제약 조건 덕분에, 레퍼런스를 사용하면  
포인터와 관련된 특정 유형의 버그를 방지할 수 있습니다:  
예를 들어, 널 포인터 간접 참조 오류와 같은 문제입니다.  
여기서 궁금할 수 있습니다: 변수 주소를 조작하기 위한  
더 안전한 메커니즘인 레퍼런스를 제공하는데, 왜 항상  
포인터 대신 레퍼런스를 사용하지 않는 걸까요?  
언어에 왜 레퍼런스와 포인터를 모두 유지할까요?  

사실, 포인터와 레퍼런스는 각자의 사용 사례가 있습니다.  
예를 들어, 포인터 산술 연산(다양한 알고리즘을 구현할 때 유용함)은  
포인터에만 적용 가능하며 레퍼런스에는 적용할 수 없습니다.  
C++ 프로그래밍을 연습하다 보면, 포인터와 레퍼런스 중  
어떤 것을 작업에 적용할지 선택하는 방법을 배우게 될 것입니다.  

다음 이론 단계에서는, C++에서 레퍼런스와 관련된 특히 중요한 사용 사례 중 하나인  
함수에 레퍼런스로 인수를 전달하는 방법에 대해 언급할 것입니다.
