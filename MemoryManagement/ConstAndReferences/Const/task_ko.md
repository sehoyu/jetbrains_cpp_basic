이 강좌에서는 이미 `const` 키워드를 다뤄보았습니다.  
이 키워드는 변경할 수 없는 변수(혹은 간단히 _상수_)를 정의할 수 있도록 합니다:  
이러한 변수에는 초기 정의 시 한 번만 값을 할당할 수 있으며, 이후에는 다시 할당할 수 없습니다.

일반적인 변경 가능한 변수와 비슷하게, 상수 변수의 주소를 가져와  
포인터 변수에 저장할 수 있습니다.  
하지만, 포인터의 타입에서 포인터가 가리키는 메모리가 상수임이 반영되어야 합니다:

```c++
const int x = 5;
const int* p = &x;
```

이와 같은 방식으로 `p`가 변경할 수 없는 메모리를 가리킨다는 것을 컴파일러에 알립니다.  
이후 만약 포인터를 이용해 상수성을 무시하려 한다면, 컴파일러는 컴파일 오류를 보고할 것입니다:

```c++
// 다음 코드는 컴파일 오류를 발생시킵니다
*p = 6;
```

마찬가지로, 상수 주소를 일반 포인터에 저장하는 것도 방지됩니다:

```c++
// 이 코드도 컴파일 오류를 발생시킵니다
int* q = &x;
```

`const int*` 타입은 상수 정수 변수(constant integer variable)를 가리키는 포인터 타입임을 유의하십시오.  
그러나 포인터 변수 자체는 상수가 아니므로 재할당할 수 있습니다:

```c++
const int y = 11;
p = &y;
```

반대로, 변경 가능한 변수에 대한 상수 포인터를 생성하는 것도 가능합니다.  
이 경우, 포인터 자체는 상수이므로 재할당할 수 없지만,  
포인터가 가리키는 변수는 여전히 재할당할 수 있습니다:

```c++
int z = 17;
int* const r = &z;
*r = 21;
// 출력값: 21
std::cout << z << "\n";
// 다음 코드는 컴파일 오류를 발생시킵니다
int w = 32;
r = &w;
```

따라서, 포인터 타입의 의미는 `const` 키워드가 `*`에 대해  
어느 위치에 놓이는지에 따라 달라집니다:
- `const`가 `*` 앞에 있다면, 이는 포인터가 가리키는 메모리의 상수성을 나타냅니다.
- `const`가 `*` 뒤에 있다면, 이는 포인터 자체의 상수성을 나타냅니다.

물론, 둘 다 결합하여 상수 변수에 대한 상수 포인터를 정의하는 것도 가능합니다:

```c++
const int* const s = &x;
// 다음 줄은 컴파일 오류를 발생시킵니다
*s = 0;
// 이 줄 또한 컴파일 오류를 발생시킵니다
s = &y;
```

다음 표는 `const` 수정자가 타입에 적용될 때의 의미를 요약합니다:

| 타입                 | 설명                                                    |
|---------------------|--------------------------------------------------------|
| `const int`         | 상수 정수                                               |
| `const int*`        | 상수 정수를 가리키는 포인터                              |
| `int* const`        | (변경 가능한) 정수를 가리키는 상수 포인터                |
| `const int* const`  | 상수 (변경 불가능한) 정수를 가리키는 상수 포인터          |
