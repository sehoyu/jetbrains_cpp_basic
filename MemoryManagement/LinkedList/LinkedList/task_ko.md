C++의 수동 메모리 관리에 대해 배운 모든 지식을 가지고,  
이제 게임으로 돌아가 새로 얻은 기술들을 실제로 적용해 보겠습니다.

기억해보면, 이전에는 고정된 크기의 정적 배열만 알고 있었기 때문에  
장면(scene)에서 소비 가능한 오브젝트의 수를 고정할 수밖에 없었습니다.  
이제 여러분의 과제는 동적 데이터 구조를 구현하여  
장면에 임의의 수의 소비 가능한 오브젝트를 배치하고,  
더 나아가서 게임 중에 새 오브젝트를 추가할 수 있게 만드는 것입니다!

여러분은 [_이중 연결 리스트_](https://en.wikipedia.org/wiki/Doubly_linked_list)라고 불리는  
기본적인 동적 데이터 구조를 구현해야 합니다.  
이 데이터 구조는 사전에 알 수 없는 임의의 수의 요소를 저장할 수 있으며,  
요소를 동적으로 추가하거나 제거할 수 있습니다.

아래의 과제 설명에서는 이중 연결 리스트를 간단히 리스트라고 부릅니다  
(일반적으로, [리스트](https://en.wikipedia.org/wiki/Linked_list)에는 다른 유형도 있습니다).  
리스트는 다음과 같은 구조체로 표현되는 _노드_로 구성됩니다:

```c++
struct Node {
    void* data;
    Node* next;
    Node* prev;
};
```

각 노드는 요소 자체에 대한 포인터와  
다음 노드 및 이전 노드를 가리키는 쌍의 포인터를 포함합니다.  
여기서 `void*`를 사용해 요소에 대한 포인터를 선언했는데,  
이는 리스트를 사용하는 사용자가 임의의 타입의 요소를 저장할 수 있도록 하고,  
타입 변환 작업을 사용자의 측면에서 처리할 수 있도록 합니다.  
간단히 말하면, 리스트의 사용자가 `malloc`을 통해  
요소 자체를 적절히 할당해야 한다고 요구합니다.

우선, 새 노드에 대한 메모리를 할당하고 해제하는 두 함수를 구현하세요:

```c++
Node* createNode(void* data);

void freeNode(Node* node);
```

`createNode` 함수는 새 노드에 대한 메모리를 할당하고,  
`data` 변수로 전달받은 값으로 `data` 필드를 초기화해야 합니다.  
`data`가 가리키는 메모리는 호출 측에서 이미 할당되었다고 가정합니다.  
`freeNode` 함수는 노드가 차지하고 있는 메모리(그리고,  
가정상 `malloc`을 통해 할당된 `data` 필드가 가리키는 메모리도 포함)를  
해제해야 합니다.

다음 구현할 함수는 `link`입니다:

```c++
void link(Node* cursor, Node* node);
```

이 함수는 `cursor`와 `node` 노드를 연결하여 `node`를 `cursor` 바로 뒤에 배치합니다.  
`cursor`, `cursor->next`, 그리고 `node` 포인터가 가리키는 관련된  
모든 노드의 `next`와 `prev` 필드를 적절하게 업데이트하도록 하세요.  
`cursor`에서 `next`와 `prev`를 통해 접근 가능한 모든 노드가 유효하며,  
그들의 `next` 또는 `prev` 포인터 중 어느 것도 null이 아니라고 가정할 수 있습니다  
(이러한 가정이 우리 리스트 구현에서 왜 참인지는 나중에 설명할 것입니다).

반대로, `unlink` 함수는 포인터로 전달받은 노드를 이웃 노드에서 분리해야 합니다:

```c++
void unlink(Node* node);
```

이 함수는 `node`의 이웃 노드들의 `next` 및 `prev` 포인터를 
서로 가리키도록 재지정해야 하며, `node`는 더 이상 가리키지 않아야 합니다.

노드 작업을 위한 몇몇 함수가 준비되었으니 이제 리스트 자체를 구현할 차례입니다.  
리스트는 여러 노드를 하나의 데이터 구조로 결합합니다.  
사실, 노드 자체가 이웃 노드에 대한 포인터를 유지하고 있기 때문에,  
리스트 데이터 구조에서 단 하나의 노드만 저장하면 됩니다.  
다른 모든 노드는 그 단일 노드를 통해 접근할 수 있습니다.

우리는 [_보초 노드_](https://en.wikipedia.org/wiki/Sentinel_node) 개념을 사용하여  
리스트를 구현하는 것을 제안합니다.  
이는 리스트 안에 항상 존재하는 특별한 노드로서,  
자체적으로 중요한 정보를 담고 있지 않으며, 가드를 역할을 합니다.  
이를 통해 첫 번째 노드를 리스트에 추가해야 할 때와 같은 경우에  
null 포인터를 다루는 것을 방지할 수 있습니다.  
이 경우 노드를 보초에 연결하기만 하면 됩니다.

```c++
struct List {
    Node sentry;
};
```

이 스킴이 제대로 작동하려면  
`sentry` 노드의 `next`와 `prev` 필드를 초기화해야 합니다.  
두 필드를 모두 `sentry` 노드 자신을 가리키도록 설정할 수 있습니다.  
따라서, 빈 리스트는 사이클을 구성하는  
`next` 및 `prev` 포인터를 가진 하나의 보초 노드로 구성된 리스트로 모델링됩니다.  
다음 아이디어를 구현하는 `initList` 함수의 코드를 작성하세요  
(`sentry` 노드의 `data` 필드를 `nullptr`로 설정하세요):

```c++
void initList(List& list);
```

다음으로, 주어진 데이터를 포함하는 새 노드를 리스트에 삽입하는 함수를 구현하세요:

```c++
void insert(List& list, void* data); 
```

그리고 마지막으로,  
특정 조건을 만족하는 데이터가 포함된 모든 노드를 제거하는 함수를 작성하세요.  
조건은 함수 포인터 형태로 제공됩니다:

```c++
void remove(List& list, bool (*pred)(void*));
```

이 함수는 리스트의 모든 노드(보초 노드는 제외)를 순회하며,  
현재 노드의 `data` 필드를 인자로 `pred` 함수를 호출해야 합니다.  
- `pred`가 `true`를 반환하면, `remove` 함수는 현재 노드를  
  리스트에서 분리하고 그 점유 메모리를 해제해야 합니다.  
  이를 위해 `unlink`와 `freeNode` 함수를 사용하세요.  
- `pred`가 `false`를 반환하면, 현재 노드를 리스트에 유지하고 다음 노드로 이동하세요.

위의 모든 함수를 올바르게 구현하면  
테스트를 모두 통과할 것입니다!  
게임 중 화면에 별 모양 소비 가능 오브젝트가  
동적으로 나타나는 것을 확인할 수 있을 것입니다!

파일 안을 살펴보며, 여러분이 구현한 리스트 조작 함수들이  
소비 가능한 오브젝트들의 동적 집합을 유지하기 위해  
어떻게 사용되고 있는지 확인해 보세요!
