C++에서 [`std::vector`](https://en.cppreference.com/w/cpp/container/vector)는 동적 배열로 동작합니다. 이 컨테이너는 일반 배열의 장점(빠른 임의 접근)을 동적 크기 조정 기능과 결합해 가장 널리 사용되는 컨테이너 중 하나입니다. 자동 메모리 관리를 사용하여 일반 배열에 비해 훨씬 좋은 대안을 제공합니다. 메모리 누수에 대한 걱정을 없애주며, 제공하는 메서드들도 사용하기 편리합니다.

`std::vector`는 순차 컨테이너이며, 크기(size) 외에도 용량(capacity)이라는 매개변수를 갖고 있습니다. 용량은 추가적인 메모리 할당 없이 컨테이너가 저장할 수 있는 요소의 개수를 의미합니다. 용량은 항상 컨테이너의 크기와 같거나 더 큽니다. 벡터에 요소를 추가할 때 현재 용량이 부족하면, 프로그램은 더 큰 용량을 가진 새로운 벡터를 생성합니다. 이 때 더 큰 크기의 배열을 새로 할당하고, 기존 배열의 모든 요소를 복사한 후, 예전 배열은 해제합니다. 할당되는 메모리의 양은 구현에 따라 다를 수 있지만, 일반적으로 현재 크기에 따라 지수적으로 증가합니다. 이런 이유로 `push_back()`과 같은 연산은 [분할 상환 상수 시간](https://en.wikipedia.org/wiki/Amortized_analysis)에 수행됩니다.

동적 메모리 할당과 해제를 방지하기 위해, 사전에 미리 저장 공간을 예약할 때 `reserve()` 메서드를 사용할 수 있습니다. 최대 크기를 미리 알고 있다면 이 방법은 프로그램의 성능을 크게 개선할 수 있습니다.

벡터는 임의의 타입에 대해 생성할 수 있습니다. 예를 들어, `std::vector<int>`는 정수 벡터이고, `std::vector<std::string>`은 문자열 벡터, `std::vector<std::vector<int>>`는 정수 벡터의 벡터(즉, 2차원 배열)입니다.

일반 배열과 비교할 때, 벡터는 배열 요소를 안전하게 참조할 수 있는 기능을 제공합니다. `at()` 메서드는 벡터의 요소에 접근할 때 사용할 수 있으며, 범위 검사를 수행하여 인덱스가 벗어나면 예외를 발생시킵니다. 하지만 범위 검사를 하지 않는 `operator[]`가 더 자주 사용되며, 이는 `at()`보다 빠르고, 다른 컨테이너들의 기능과도 일치하기 때문입니다.

벡터를 생성하면서 요소 값을 명시하지 않으면, 각 요소는 기본값으로 초기화됩니다. 예를 들어, `std::vector<int> v(3)`는 세 개의 정수로 구성된 벡터를 만들고, 각 요소는 0으로 초기화됩니다. 초기값을 지정할 수도 있습니다. 예를 들어, `std::vector<int> v(3, 1)`은 세 개의 정수로 이루어진 벡터를 만들며, 각 요소는 1로 초기화됩니다.

벡터는 여러 가지 방법으로 생성할 수 있습니다:

```cpp
// 빈 벡터 생성
std::vector<int> v1;

// 요소를 하나씩 추가
v1.push_back(1);
v1.push_back(2);
v1.push_back(3);
// v1 = {1, 2, 3}

// 지정된 크기와 값으로 벡터 생성 및 초기화
std::vector<int> v2(3, 1); // v2 = {1, 1, 1}

// 기존 배열로부터 벡터 생성
int a[] = {1, 2, 3};
std::vector<int> v3(a, a + 3); // v3 = {1, 2, 3}

// 이니셜라이저 리스트를 이용한 벡터 생성
std::vector<int> v4 = {1, 2, 3}; // v4 = {1, 2, 3}

// 크기만 지정하여 기본값으로 초기화된 벡터 생성
std::vector<int> v5(3); // v5 = {0, 0, 0}
```

`std::vector`의 주요 메서드는 다음과 같습니다:
 - `size()`: 벡터의 요소 개수를 반환합니다.
 - `capacity()`: 벡터의 용량을 반환합니다.
 - `empty()`: 벡터가 비어 있는지 확인합니다.
 - `clear()`: 벡터의 모든 요소를 제거합니다.
 - `push_back(x)`: 요소를 벡터의 끝에 추가합니다.
 - `emplace_back(x)`: 벡터 끝에 요소를 직접 생성합니다. 복사 및 이동 연산을 피하므로 `push_back()`보다 더 빠르지만, 해당 요소가 이미 생성되어 있지 않을 때만 사용할 수 있습니다.
 - `pop_back()`: 벡터의 마지막 요소를 제거합니다.
 - `resize(x)`: 벡터의 크기를 변경합니다. 크기를 늘릴 경우 새 요소는 주어진 값(기본값은 0)으로 초기화됩니다. 크기를 줄이면 새 크기 이후의 모든 요소가 제거됩니다.
 - `swap(a, b)`: 두 벡터의 내용을 교환합니다.
 - `front()`, `back()`: 각각 벡터의 첫 번째와 마지막 요소를 반환합니다.

또한, 벡터는 임의 위치에 요소 삽입 및 삭제를 지원합니다. 주요 메서드는 다음과 같습니다:
- `insert()`: 단일 요소 또는 요소의 범위를 벡터에 삽입합니다. 삽입 지점 이후 모든 요소를 이동해야 하므로 속도가 느리며, 선형 복잡도를 가집니다.
- `erase()`: 단일 요소 또는 요소의 범위를 벡터에서 제거합니다. 동작 방식은 `insert()`와 동일함으로, 사용 시 주의가 필요합니다.

`std::vector` 및 많은 요소로 구성될 수 있는 다른 모든 컨테이너의 경우, 함수에 인자로 전달할 땐 보통 참조(reference)로 전달하는 것이 좋습니다. 참조로 전달하지 않을 경우, 벡터 전체가 복사되어 전달됩니다. 요소가 많은 컨테이너에서는 이 복사가 성능에 큰 영향을 줄 수 있습니다.

```cpp
void print(const std::vector<int>& v) {
    for (int i = 0; i < v.size(); ++i) {
        std::cout << v[i] << ' ';
    }
    std::cout << '\n';
}
```
단, 모든 상황에 해당되는 것은 아닙니다. 만약 컨테이너를 수정해야 한다면, 값으로 전달(이 경우, 호출자에게 변경사항이 반영되지 않음)하거나, 비-const 참조로 전달(초기 컨테이너를 수정)할 수 있습니다. 자세한 내용은 [이 글](https://belaycpp.com/2022/02/15/constant-references-are-not-always-your-friends/)을 참고하십시오.

여러분의 과제는 다음과 같은 멤버 함수들을 사용하는 여러 함수를 구현하는 것입니다:
 - `create()`는 정수형 벡터를 생성하고, 매개변수로 주어진 개수만큼의 요소를 저장할 공간을 미리 예약해야 합니다.
 - `fill()`은 1부터 10까지의 값을 값으로 갖는 10개의 요소로 벡터를 채워야 합니다.
 – `debugPrint()`는 벡터의 크기, 용량, 그리고 모든 요소를 표준 출력에 출력해야 합니다. 출력 형식은 다음과 같습니다:
   - `Size: <n>, Capacity: <n>`
   - `Vector: <element1> <element2> ... <elementN>`
