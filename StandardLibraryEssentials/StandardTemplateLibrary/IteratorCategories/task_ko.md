C++에서 반복자(iterator)는 그 기능과 동작 방식에 따라 여러 범주로 분류됩니다. 각 범주를 이해하는 것은 중요하며, 다양한 STL 컨테이너들은 각각 서로 다른 반복자 범주를 제공합니다. 반복자 범주는 다음과 같습니다:
1. **입력 반복자(Input Iterators)**: 참조하는 요소로부터 값을 읽는 작업을 지원합니다. 한 번 값을 읽은 후, 반복자를 증가시켜 다음 요소를 가리킬 수 있습니다.
2. **출력 반복자(Output Iterators)**: 참조하는 요소에 값을 쓰는 작업을 지원합니다. 값을 기록한 후, 반복자를 증가시켜 다음 요소를 가리킬 수 있습니다.
3. **전방 반복자(Forward Iterators)**: 입력 및 출력 반복자의 특징을 모두 갖추고 있습니다. 여러 번 범위를 통과할 수 있으며, 읽기 및 쓰기 작업을 모두 지원합니다.
4. **양방향 반복자(Bidirectional Iterators)**: 전방 반복자에 더해 `--` 연산자를 사용하여 범위를 뒤로 이동할 수 있는 기능을 제공합니다.
5. **임의 접근 반복자(Random Access Iterators)**: 가장 다양한 기능을 제공하며, 범위 내의 모든 요소에 상수 시간에 접근할 수 있고, 산술 연산 및 포인터와 유사한 동작을 지원합니다.

아래는 반복자 범주에 대한 간략 비교 표입니다:

| 범주          | 읽기 | 쓰기 | 전방 이동 | 후방 이동 | 임의 접근 |
|---------------|------|-------|-----------|-----------|-----------|
| 입력          | 예   | 아니오| 예        | 아니오    | 아니오    |
| 출력          | 아니오| 예    | 예        | 아니오    | 아니오    |
| 전방          | 예   | 예    | 예        | 아니오    | 아니오    |
| 양방향        | 예   | 예    | 예        | 예        | 아니오    |
| 임의 접근     | 예   | 예    | 예        | 예        | 예        |

반복자는 소속된 컨테이너에서 특정 작업이 수행될 때 무효화될 수 있습니다. 반복자 무효화(Iterator Invalidation)를 이해하는 것은 정의되지 않은 동작을 피하기 위해 매우 중요합니다. 일반적인 반복자 무효화 상황은 다음과 같습니다:
- **삽입 및 삭제**: 컨테이너에 요소를 삽입하거나 삭제하면, 해당 요소를 가리키고 있던 반복자가 무효화될 수 있습니다.
- **컨테이너 크기 변경**: 크기 조정 작업은 내부 메모리 재할당을 유발할 수 있으며, 이로 인해 모든 반복자가 무효화될 수 있습니다.
- **컨테이너 비우기**: 컨테이너를 비우면 그 요소들을 가리키던 모든 반복자가 무효화됩니다.

추가 사례는 [이 글](http://kera.name/articles/2011/06/iterator-invalidation-rules-c0x/)을 참고하시기 바랍니다.

각 STL 컨테이너마다 제공하는 반복자 범주와 반복자 무효화 규칙이 다릅니다. 아래 표는 각 STL 컨테이너별 반복자 범주와 함께, 각 컨테이너에 해당하는 반복자 무효화 규칙을 정리한 것입니다:

| 컨테이너      | 반복자 범주       | 반복자 무효화 규칙                |
|---------------|-------------------|----------------------------------|
| vector        | 임의 접근         | 삽입, 삭제, 크기 변경             |
| string        | 임의 접근         | 삽입, 삭제, 크기 변경             |
| list          | 양방향            | 없음                              |
| deque         | 임의 접근         | 양 끝에서의 삽입, 삭제            |
| stack         | 해당 없음         | 해당 없음                         |
| queue         | 해당 없음         | 해당 없음                         |
| set           | 양방향            | 삽입, 삭제                        |
| map           | 양방향            | 삽입, 삭제                        |
| unordered_set | 전방              | 삽입, 삭제, 재해싱                |
| unordered_map | 전방              | 삽입, 삭제, 재해싱                |

stack과 queue는 반복자를 제공하지 않음을 유의하시기 바랍니다. 반복자 범주에 대한 더 자세한 내용 및 전체 표는 [C++ reference](https://en.cppreference.com/w/cpp/iterator)에서 확인하실 수 있습니다.
