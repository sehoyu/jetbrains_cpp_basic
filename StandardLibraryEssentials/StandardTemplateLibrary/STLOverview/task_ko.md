이 모듈에서는 모든 C++ 프로그래머의 조력자인 표준 템플릿 라이브러리(STL)에 대해 깊이 알아보겠습니다! 기본 언어 기능 외에도, 모든 C++ 컴파일러는 미리 작성된 다양한 알고리즘, 자료구조, 그리고 기타 일반화 클래스와 함수에 접근할 수 있도록 제공합니다.

다음 주제를 다룰 예정입니다:
- `std::vector` 기능
- `std::string` 및 해당 최적화
- `std::forward_list`, `std::deque`, `std::list`, `std::array`와 같은 순차 컨테이너
- `std::stack`, `std::queue`, `std::priority_queue`와 같은 순차 컨테이너 어댑터
- `std::set`, `std::map`과 같은 연관 컨테이너
- `std::unordered_set`, `std::unordered_map`과 같은 비순차 연관 컨테이너
- 반복자와 각 반복자 유형
- STL 알고리즘
- 입출력 스트림

STL은 수백 개의 클래스와 함수로 이루어져 있어, 한 과정에서 모두 다루는 것이 불가능합니다. 그러나 가장 중요한 요소들을 소개하고, 나머지를 스스로 학습할 수 있는 도구도 제공할 예정입니다. [Cppreference](https://en.cppreference.com/w/)는 STL과 C++ 언어 전반에 대해 자세한 정보를 제공하는 훌륭한 자료입니다.

가끔 우리는 연산 또는 알고리즘의 "복잡도"에 대해 언급합니다. 복잡도란 한 연산이나 알고리즘이 완료되는 데 소요되는 시간과 메모리의 양을 측정한 것입니다. 우리는 이 복잡도를 설명할 때 [빅오 표기법](https://en.wikipedia.org/wiki/Big_O_notation)을 사용할 것입니다. 예를 들어, 어떤 연산이 `O(n)`의 시간이 걸린다면, 이는 완료하는 데 필요한 시간이 입력 크기에 비례함을 의미합니다. `O(1)` 또는 "상수 시간"은 해당 연산의 시간이 입력 크기에 관계없이 일정하다는 뜻입니다.
