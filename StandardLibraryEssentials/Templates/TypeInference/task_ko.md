타입 추론(Type inference)은 C++에서 컴파일러가 변수나 표현식의 타입을 컴파일 타임에 자동으로 유추하도록 해주는 강력한 기능입니다. 이를 통해 코드가 더 간결하고 읽기 쉬워질 뿐만 아니라, 타입과 관련된 오류의 가능성도 줄일 수 있습니다. 이름이 복잡하거나 생소한 타입을 다룰 때도 타입 추론을 활용하면 코드의 가독성과 유지보수성이 크게 향상됩니다.

타입 추론은 파이썬이나 자바스크립트 같은 언어의 특징인 동적 타이핑(dynamic typing)과는 다르다는 점에 유의해야 합니다. C++에서 변수의 타입은 한 번 결정되면, 해당 변수가 소멸될 때까지 변경되지 않습니다.

타입 추론은 컴파일 타임에 변수나 표현식의 타입이 사용 맥락에 따라 결정되는 기능입니다.

C++에서 타입 추론은 주로 `auto` 키워드를 통해 이루어집니다. `auto`를 사용하면 변수나 표현식의 초기값을 바탕으로 컴파일러가 타입을 자동으로 유추하게 됩니다. 예를 들어, 타입을 명시하지 않고 변수를 선언할 때 `auto`를 사용할 수 있습니다:
```cpp
auto x = 42; // x는 int로 유추됩니다
auto y = 3.14; // y는 double로 유추됩니다
auto str = "Hello, World!"; // str은 const char*로 유추됩니다
```

경우에 따라 `auto`를 사용해 참조 변수를 선언하고 싶을 때가 있습니다. 이 경우에는 `auto&`를 사용하여 변수임을 명확히 지정해야 합니다:
```cpp
int a = 42;
int& ref = a;
auto b = ref; // b는 int로 유추됩니다
auto& ref2 = ref; // ref2는 int&로 유추됩니다
```

C++11에서 새롭게 도입된 키워드 중 하나는 `decltype`입니다. `decltype`을 사용하면 표현식을 실제로 평가하지 않고도 해당 표현식의 타입을 유추할 수 있습니다. 이는 다른 표현식과 동일한 타입으로 변수를 선언해야 할 때 유용합니다:
```cpp
int a = 42;
decltype(a) b = 3 + a; // b는 int로 유추됩니다
```

함수 및 클래스 템플릿에서도, 컴파일러는 종종 전달된 인자의 타입을 바탕으로 템플릿 인자 타입을 자동으로 유추할 수 있습니다. 따라서 함수 템플릿을 호출할 때 대부분 템플릿 인자를 명시적으로 지정하지 않아도 되는데, 이를 _템플릿 인자 추론_(template argument deduction)이라고 합니다. 사용 예시는 다음과 같습니다:
```cpp
template <typename T>
void print(T value) {
    std::cout << value << std::endl; 
} 

int main() { 
    print(10); // print<int>(10)이 호출됩니다
    print(3.14); // print<double>(3.14)가 호출됩니다
    print("Hello, World!"); // print<const char*>("Hello, World!")가 호출됩니다
    return 0;
}
