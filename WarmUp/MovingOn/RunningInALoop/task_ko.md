장면에 소비 가능한 객체를 더 추가해 봅시다!

동일한 타입의 여러 변수를 [_배열_](https://en.cppreference.com/w/cpp/language/array)에 저장할 수 있습니다.

```c++
const int SIZE = 4;
Circle circles[SIZE];
```

위 코드 스니펫은 `Circle` 타입의 구조체 4개를 포함하는 배열을 정의합니다.  
배열의 `i`번째 요소에 접근하려면 대괄호 문법을 사용하세요.  
인덱스는 `0`부터 시작하며, 즉 배열의 첫 번째 요소는 인덱스 `0`을 가집니다.

```c++
// 배열에서 첫 번째 원의 반지름을 다시 할당합니다
circles[0].radius = 2.0f;
```

배열 작업 시, 반복적으로 수행되는 작업 중 하나는 배열의 모든 요소를 순회(iterate)하며  
각 요소에 대해 계산을 수행하는 것입니다.  
여기서 `for` [문](https://en.cppreference.com/w/cpp/language/for)이 이를 해결해줍니다.

```c++
// 배열에 있는 모든 원의 반지름을 출력
for (int i = 0; i < SIZE; ++i) {
    std::cout << circles[i] << " ";
}
```

`for` 반복문은 다음과 같은 구성 요소로 이루어져 있습니다:
- 초기화 문: `int i = 0`,
- 조건식: `i < SIZE`,
- 반복 식: `++i`,
- 반복문의 본문: `std::cout << circles[i] << " ";`.

`for` 반복문의 실행은 다음과 같이 진행됩니다:
- 반복문 실행 시, 초기화 문이 실행됩니다.
- 이후, 각 반복에서 조건식이 평가됩니다:
  - 조건식이 `true`로 평가되면, 반복문의 본문이 실행됩니다.
  - 조건식이 `false`로 평가되면, 반복문이 종료됩니다.
- 각 반복에서 반복문의 본문 실행이 끝나면 반복 식이 실행되고 다음 반복이 시작됩니다.

위 코드에서 [증가 연산자](https://en.cppreference.com/w/cpp/language/operator_incdec) `++i`를 사용했습니다.  
이 표현식은 변수의 값을 `1` 증가시키고, 증가된 값을 반환합니다.

```c++
int x = 0;
int y = ++x;
// "1, 1" 출력
std::cout << x << ", " << y;
```

마찬가지로 감소 표현식도 존재합니다: `--i`.

```c++
int x = 0;
int y = --x;
// "-1, -1" 출력
std::cout << x << ", " << y;
```

증가 및 감소 표현식은 전위(pre-) 또는 후위(post-) 형식으로 사용할 수 있습니다.  
위에서는 전위 증가(및 전위 감소)를 보았습니다: 변수의 값을 먼저 증가시키고  
그 값을 반환합니다.  
후위 증가(및 후위 감소)는 `x++`(또는 `x--`)로 작성하며, 변수의 값을 증가시키지만 수정 이전의 값을 반환합니다.

`for` 반복문에서 조건식과 반복 식은 임의로 복잡하게 작성할 수 있습니다.  
이것은 `for` 반복문이 매우 유연하게 작동할 수 있음을 의미합니다.  
예를 들어, 배열에서 짝수 인덱스를 가진 요소를 순회하려면 아래 코드를 사용할 수 있습니다:

```c++
// 배열의 짝수 인덱스를 가진 모든 원의 반지름을 출력
for (int i = 0; i < SIZE; i += 2) {
    std::cout << circles[i] << " ";
}
```

위의 이론을 바탕으로  
다음 과제는 플레이어 객체와 화면 상의 모든  
소비 가능한 객체의 충돌 여부를 확인하는 것입니다.  
이를 위해 다음 함수를 구현하십시오:

```c++
void collisionLoop(Circle player, Circle consumable[], bool consumed[], int size);
```

이 함수는 플레이 가능한 객체의 원 형태와  
소비 가능한 객체들의 배치 배열을 함께 가져옵니다.  
그리고 그 상태를 나타내는 불리언 배열도 같이 받습니다:
- `consumed[i] == true`는 `i`번째 객체가 소비되었음을 의미합니다.
- `consumed[i] == false`는 `i`번째 객체가 아직 소비되지 않았음을 의미합니다.

두 배열의 크기는 동일하며,  
마지막 인수로 주어진 장면 내 소비 가능한 객체의 총 개수와 같습니다.

이 함수는 플레이 가능한 객체가  
소비 가능한 객체들과 충돌하는지 확인해야 합니다.  
충돌한 경우, 해당 객체의 상태를 "소비됨(consumed)"으로 설정해야 합니다.  
한번 소비된 객체는 다시 "소비되지 않음(non-consumed)" 상태로 변경될 수 없습니다.  
충돌을 감지하려면, 이전 단계에서 사용한 `collision` 함수를 사용하십시오.
